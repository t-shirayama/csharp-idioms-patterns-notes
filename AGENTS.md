# AGENTS.md

このリポジトリは、実務で役立つ C# のイディオム、定石、設計パターン、落とし穴を日本語で整理するノートです。

## 作業方針

- 日本語で、実務経験者が読み返して判断に使える内容を書く。
- 単なる文法説明ではなく、使いどころ、避けたい書き方、レビュー観点、テストしやすさを含める。
- C# 8 以降の現代的な書き方を中心にする。
- 古い書き方も、現場で遭遇しやすいものは比較対象として扱う。
- サンプルコードは短く、意図が分かる単位にする。
- 「よい例」だけでなく「避けたい例」も載せる。
- パフォーマンスの話は、必要な場面と過剰最適化の境界を明示する。

## README の扱い

- README.md はリポジトリ全体の目次とロードマップとして扱う。
- 章構成を変更する場合は、全体の流れが崩れないか確認する。
- 章を追加する場合は、既存の Part / Chapter 形式に合わせる。
- README に長い本文を書きすぎず、将来的に章別 Markdown へ分割できる粒度を保つ。

## 文章スタイル

- 読者は「C# の基本文法は分かるが、実務での書き方に迷う人」を想定する。
- 断定しすぎず、判断基準を添える。
- 抽象論だけで終わらせず、できるだけ C# コード上の具体例に結びつける。
- 用語は必要に応じて英語を併記する。
- 見出しは簡潔にし、箇条書きは読み返しやすさを優先する。

## コード例の方針

- C# のコードブロックには `csharp` を指定する。
- 不要に大きなクラスやアプリケーション全体の例は避ける。
- 例外、null、非同期、LINQ、コレクション操作では、失敗しやすいケースも示す。
- テストしやすさを扱う場合は、依存の分離や境界の置き方を明確にする。

## 変更時の確認

- Markdown の見出し階層が崩れていないか確認する。
- README の見出しや Part 構成に重複や不整合がないか確認する。
- 新しい項目が、このリポジトリの目的から外れていないか確認する。
- 既存のユーザー作業や未コミット変更を不用意に戻さない。
