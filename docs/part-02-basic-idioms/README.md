# Part 2: 基本文法の実務イディオム

基本文法は、書けるだけならすぐに通過できる領域です。実務では、同じ構文でも「どこで境界を作るか」「読み手が失敗条件を追えるか」「テストしやすい形になっているか」で保守性が大きく変わります。

この Part では、C# の基本構文をレビューで判断しやすい単位に整理します。
