# Part 10: パフォーマンスと落とし穴

パフォーマンス改善は、速そうな書き方を選ぶ作業ではなく、遅さの原因を測り、影響が大きい場所から直す作業です。実務では、allocation、GC、文字列処理、LINQ、非同期、例外処理などが「読みやすさ」と「実行時コスト」の境界になりやすいです。

この Part では、日常のコードレビューで見つけやすい落とし穴を中心に扱います。最初から低レベル最適化へ寄せるのではなく、hot path かどうか、入力サイズが増えるか、失敗時にどう振る舞うかを確認してから判断します。

性能上の指摘は、正しさ、可読性、テストしやすさとの交換になります。測定が必要な箇所と、読みやすさを優先してよい箇所を分けて、チームで同じ基準を持てるようにします。
